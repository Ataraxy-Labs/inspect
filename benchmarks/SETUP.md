# Martian Code Review Benchmark — Setup Guide

End-to-end setup to reproduce the inspect benchmark results.

## Prerequisites

- macOS or Linux
- ~10GB free disk space
- GitHub token (for cloning repos + API calls)
- OpenAI API key or Martian API key (for the GPT-5.2 judge)
- Anthropic API key (for running Claude Opus agent reviews)
- Node.js 20+ with `tsx` installed (`npm i -g tsx`)
- Python 3.12+
- `ast-grep` CLI (`brew install ast-grep` or `cargo install ast-grep`)
- inspect binary built: `cargo build --release` from repo root

## Step 1: Clone the Martian benchmark repo

```bash
git clone https://github.com/withmartian/code-review-benchmark.git /tmp/martian-eval/benchmark

# The "full" version has additional data — clone it to a second location
# (or symlink if you only have one copy)
cp -r /tmp/martian-eval/benchmark /tmp/martian-eval/code-review-benchmark-full
cp -r /tmp/martian-eval/benchmark /tmp/martian-eval/code-review-benchmark
```

Install the judge dependencies:
```bash
cd /tmp/martian-eval/code-review-benchmark-full/offline
uv sync  # or: pip install -e .
```

## Step 2: Set up API keys

Create `/tmp/martian-eval/code-review-benchmark-full/offline/.env`:
```bash
# For the GPT-5.2 judge (via Martian or OpenAI directly)
MARTIAN_API_KEY=your-key-here
MARTIAN_BASE_URL=https://api.withmartian.com/v1
MARTIAN_MODEL=router-default
```

Create `~/Documents/work/inspect/.env`:
```bash
ANTHROPIC_API_KEY=sk-ant-...
OPENAI_API_KEY=sk-...
GITHUB_TOKEN=ghp_...
```

## Step 3: Clone the 5 source repos (blobless clones, ~2GB total)

The benchmark covers 50 PRs across these repos:

```bash
mkdir -p /tmp/martian-eval/repos

# Blobless clones (fast, ~400MB each instead of multi-GB)
git clone --no-checkout --filter=blob:none https://github.com/getsentry/sentry.git /tmp/martian-eval/repos/getsentry__sentry
git clone --no-checkout --filter=blob:none https://github.com/grafana/grafana.git /tmp/martian-eval/repos/grafana__grafana
git clone --no-checkout --filter=blob:none https://github.com/keycloak/keycloak.git /tmp/martian-eval/repos/keycloak__keycloak
git clone --no-checkout --filter=blob:none https://github.com/discourse/discourse.git /tmp/martian-eval/repos/discourse__discourse
git clone --no-checkout --filter=blob:none https://github.com/calcom/cal.com.git /tmp/martian-eval/repos/calcom__cal.com

# Also clone the greptile evaluation fork for sentry (has 2 PRs with different golden comments)
git clone --no-checkout --filter=blob:none https://github.com/ai-code-review-evaluation/sentry-greptile.git /tmp/martian-eval/repos/ai-code-review-evaluation__sentry-greptile
```

## Step 4: Create worktrees for all 50 PRs

Each PR needs a git worktree checked out at the PR's head SHA. The easiest way
is to run `martian_eval.py` which does this automatically:

```bash
cd ~/Documents/work/inspect

# This will:
# 1. Load golden comments from the Martian benchmark
# 2. Resolve base/head SHAs via GitHub API (cached in /tmp/martian-eval/sha_cache/)
# 3. Fetch commits into each repo
# 4. Create git worktrees in /tmp/martian-eval/worktrees/{repo}/{base_head}/
# 5. Run inspect's AST analysis on each PR
python3 benchmarks/martian_eval.py --mode ast --limit 1
```

This creates one worktree as a test. To create all 50:

```bash
python3 benchmarks/martian_eval.py --mode ast
```

The worktrees are cached — subsequent runs reuse them. Total: ~6GB for all 50 worktrees.

## Step 5: Generate the PR data file

The benchmark harness (`run_diff_bench_v4.py`) reads from a pre-generated
results file that maps PR numbers to base/head SHAs:

```
benchmarks/results/20260318_183258_ast.json
```

This was generated by an earlier `martian_eval.py --mode ast` run. If you need
to regenerate it, run the ast mode and update the `OUR_RESULTS` path in
`run_diff_bench_v4.py`.

## Step 6: Build inspect and install agent dependencies

```bash
# Build the Rust binary
cargo build --release

# Install agent Node.js dependencies
cd agent && npm install && cd ..
```

## Step 7: Run the full benchmark

```bash
python3 benchmarks/run_benchmark.py --concurrency 47
```

This runs two steps:
1. **Agent reviews** (~10-15 min): Runs `agent/src/review-diff-v2.ts` on all 47
   PRs in parallel. Each agent gets the diff + inspect's entity analysis, uses
   Claude Opus with tool access (read, grep, bash, ast-grep, inspect) to review,
   then extracts bugs from the review.

2. **Martian judge** (~3 min): Maps our results to golden URLs, builds
   `candidates.json` with the full raw review text per PR, then runs the
   Martian `step3_judge_comments.py` which uses GPT-5.2 to score TP/FP/FN.

Results are printed as F1/Precision/Recall plus a leaderboard comparison.

### Useful flags

```bash
# Run specific PRs only
python3 benchmarks/run_benchmark.py --pr 37429 8330

# Re-judge without re-running agents (uses latest summary)
python3 benchmarks/run_benchmark.py --judge-only

# Judge a specific run
python3 benchmarks/run_benchmark.py --judge-only --summary benchmarks/results/diff_bench_logs/20260325_140941/_summary_v7.json

# Just view existing results
python3 benchmarks/run_martian_judge.py --results-only
```

## Cost estimate per run

- **Agent reviews**: ~47 Claude Opus calls with tools × ~$3-5 each ≈ **$150-250**
- **Judge**: ~47 GPT-5.2 calls × ~$0.10 each ≈ **$5**
- **Total**: ~$150-250 per full run

## Known caveats

1. **Blob mode scoring**: We submit the full raw review text as a single candidate
   per PR. Other tools on the leaderboard are scored on individual extracted
   comments. This means the judge has more text to find matches in, which likely
   inflates our recall relative to tools with granular per-line comments.

2. **Training data leakage**: These are well-known open-source PRs. Claude may
   have seen them during training. The Martian benchmark acknowledges this.

3. **Run-to-run variance**: F1 varies ±5 points across runs (59-73% observed)
   due to LLM non-determinism in both the review agent and the judge.

4. **3 PRs are excluded**: sentry-greptile/pull/5 (no original_url mapping),
   grafana/pull/90939 (no worktree), sentry/pull/94376 (>100s graph build).
   We evaluate 47 of 50 PRs.
